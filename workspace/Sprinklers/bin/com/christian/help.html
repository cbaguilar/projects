Possible Errors:

Network Problems:

If you encounter a "Connection Refused" error when trying to connect to the Raspberry Pi, the Raspberry Pi server program is probably not currently on or functioning.

If you have a "No route to host" error, either the specified IP address of the Raspberry Pi is incorrect, the Pi is not connected to the network, or you are not connected to the network.

As of this update, the IP address of Pi is "192.168.2.8". If that is incorrect, check to see that the Pi is running (Indicated by the LEDs")
